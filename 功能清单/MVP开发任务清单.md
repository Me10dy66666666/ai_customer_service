# AI智能客服 MVP开发任务清单

根据《AI智能客服系统技术方案设计.md》拆解的MVP（最小可行性产品）开发任务清单。
本清单严格遵循**"知识库搭建 → 多渠道问答 → 用户权限区分 → 咨询记录分析 → 工单流转"**的业务闭环，并按优先级进行了排序。

## 一、 MVP开发任务清单

## 三、 技术实现说明

### 1. 游客身份管理
- 游客通过前端生成的 `sessionId` (格式: `guest-随机字符串-时间戳`) 进行标识
- 会话ID存储在浏览器 `sessionStorage` 中，确保同一会话内身份一致性
- 后端通过 `consultation_logs.session_id` 字段关联游客的所有对话记录

### 2. AI集成架构
- 使用 Dify 作为 RAG 和 LLM 服务提供商
- 知识库文档上传到 Dify 向量库进行检索
- 对话通过 Dify Chat API 进行流式处理
- 异常场景（意图识别失败、知识库无匹配等）通过 JSON 结构化输出触发工单创建

### 3. 权限控制
- 注册用户：通过 JWT Token 认证，userType = 1
- 游客：无 Token，通过 sessionId 识别，userType = 0
- 管理员：userType = 3，可访问管理功能
- 工单创建：仅限注册用户，游客尝试创建工单会提示先登录

### 4. 异常处理场景
- **FR-022 意图识别失败**: 置信度 < 60% 时自动转人工
- **FR-023 知识库无匹配**: 自动转人工并创建知识库补充工单
- **FR-024 报价计算失败**: 转人工销售提供精准报价
- **FR-025 复杂问题**: AI指导2次未解决时创建售后工单
- **FR-026 产品质量故障**: 自动创建质量故障工单
